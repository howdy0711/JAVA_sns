원격네트워크에 도달하는 방법


정적 - 수동으로 고정된 경로를 사용하여 도달
장점  
1. 다른네트워크를 거지치 않아 동적라우팅보다 조금 더 보안성이 좋다.
2. 경로설정을 위해서 CPU를 사용하지 않음으로 CPU부하가 적다.
3. 데이터를 보내는데 사용하는 경로가 알려져 있다. 
단점
1. 초기구성 및 유지보수 하는데 시간이 오래걸린다.
2. 대규모 네트워크에서 오류가 발생하기 쉽다.
3. 경로를 변경할 때마다 관리자의 개입이 필요하다.
4. 확장성이 떨어지며 네트워크가 커질수록 설정하기 번거롭다.
정적사용목적:
더이상 커질 것 같지 않는 소규모 네트워크
스텁 네트워크 환경에서의 라우팅
해당경로가 없을 경우 디폴트 라우트 설정

동적 - 자동적으로 동적 라우팅 프로토콜을 사용하여 학습



기본정적 경로는 모든 패킷이 일치하는 루트이다.
라우터에서 어떤 목적지까지 경로가, 라우팅 테이블에 없는 경우 기본적으로
또는 최종적으로 어쩔수 없이 선택되어지는 디폴트 경로이다.

ip route 0.0.0.0 0.0.0.0 {넥스트홉ip]또는{출구 인터페이스}
넥스트홉으로 입력할경우 리커시브 룩업이 일어나, 전송속도가 느려진다.
출구 인터페이스를 사용할 경우 바로 전송이된다. 하지만 출구인터페이스를 
사용하지 못하는경우도 있다. 점대점 방식이 아닌 멀티엑세스형태에서는 어느 
인터페이스로 갈지를 정해주어야한다. 이럴때에는 반드시 넥스트홉 ip주소를 적어
주어야한다.

fully specfied static route 나가는 인터페이스와 nexthop ip를 모두 지정한다.


요약 정적 라우트
다른 네트워크 대역을 하나로 묶는 방법을 말한다.

172.20.0.0 /16
172.21.0.0 /16
172.22.0.0 /16
172.23.0.0 /16

는 172.20.0.0 /14로 묶어 질수 있다.

Floating 정적 라우트
링크 장애가 발생한 경우 주요 정적또는 동적경로에 대한 백업 길을 제공하기 위해서
사용되는 정적 경로이다. 이러한 floating static route는 높은 ad값으로 설정된다.
